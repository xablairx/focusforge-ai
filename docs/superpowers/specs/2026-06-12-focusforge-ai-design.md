# FocusForge AI — Design Spec
**Date:** 2026-06-12
**Status:** Approved
**Working title:** FocusForge AI

---

## Overview

A SaaS web application that acts as an AI Chief of Staff for entrepreneurs, freelancers, solopreneurs, ADHD professionals, overwhelmed creators, and stay-at-home parents. The core problem: users constantly chase new ideas, start new projects, buy courses, redesign websites, tweak logos, and plan endlessly instead of executing revenue-generating activities.

FocusForge AI prevents users from sabotaging themselves through distraction. It keeps them locked onto one mission long enough to achieve meaningful results.

---

## Core Philosophy

- **Execution over planning**
- **Revenue over busywork**
- **Completion over perfection**
- **Focus over endless options**

The AI behaves like an executive coach and accountability partner — direct, honest, occasionally challenging — not a cheerleader, therapist, or motivational quote machine.

---

## Target Personas

1. **Freelancer / solopreneur** — trying to land first client or scale revenue
2. **ADHD professional** — high idea volume, low follow-through, needs external structure
3. **Overwhelmed creator** — YouTube, newsletter, course creator with too many active projects
4. **SAHM (stay-at-home mom)** — limited time windows (30–90 min), needs maximum leverage per minute
5. **Entrepreneur in early stage** — pre-revenue, building an MVP or portfolio

## Features

### 1. Single Active Mission

Users can have only one active mission at a time. Examples: "Get first client", "Launch MVP", "Complete certification", "Lose 20 pounds", "Create portfolio".

- Mission is created during onboarding and stored in the `missions` table
- One-active-mission constraint enforced at the DB level via partial unique index: `UNIQUE (user_id) WHERE status = 'active'`
- All AI recommendations are anchored to the active mission
- Mission tracks: days on mission, completion percentage (manually set by user via slider, 0–100%), status

### 2. Idea Jail

Users can submit unlimited ideas at any time. The AI automatically assesses whether each idea supports the current mission.

- Ideas are scored 0–10 for mission alignment by Claude (structured output)
- Score 0–4 → **Jailed**: locked until mission completes or weekly review
- Score 5–7 → **Flagged**: surfaced at weekly review for consideration
- Score 8–10 → **Approved**: directly supports mission, user prompted to add as a task
- Ideas from past missions are accessible via a "Past Missions" filter tab on the Idea Jail page
- "Jail Idea" quick action available from the dashboard

### 3. AI Chief of Staff — Daily Check-in

Each day (once per day, gated by route guard), the user is prompted with 3 questions:
1. How many hours/minutes are you available today?
2. What obstacles exist?
3. What progress did you make yesterday?

Claude generates a structured response (zod-validated):
- 1 primary task (revenue score 8–10)
- Up to 2 secondary tasks (revenue score 4–7)
- 1 coaching note
- Maximum 3 tasks per day, no exceptions

**SAHM Mode override:** If available time < 1 hour, Claude returns exactly 1 task — the single highest-leverage action completable in the available window. Coaching note acknowledges the time constraint without guilt or judgment.

### 4. Revenue Scoring Engine

Every task generated by the AI receives a revenue score (0–10) reflecting its direct impact on mission revenue:

| Task Type | Score |
|-----------|-------|
| Direct client outreach | 10 |
| Sales call | 10 |
| Proposal / contract sent | 9 |
| Follow-up on active deal | 9 |
| Lead magnet / portfolio update | 6 |
| Course lesson / skill-building | 4 |
| Logo redesign | 1 |
| Social media scrolling | 0 |

Scores are assigned by Claude at check-in time based on task description and mission context. The dashboard always shows the highest-score task first. Revenue score is stored on the `tasks` table.

### 5. Anti-Shiny Object Protocol

Triggered when:
- User attempts to change or create a new mission
- User clicks "Abandon Mission"
- User submits an idea that scores ≥ 8 and resembles a pivot
- Mission age < 14 days + any of the above

Claude delivers a direct challenge via the chat interface (streaming):

> *"You've been on this mission for 8 days. Before you switch, answer honestly: is this mission no longer valid — or has executing on it become uncomfortable? Because there's a difference between a dead end and resistance. One requires a pivot. The other requires you to push through."*

User is presented with two outcomes:
- **"I want to switch anyway"** → mission marked `abandoned`, reason logged, new mission flow begins
- **"You're right, I'll continue"** → returned to dashboard, idea is jailed

Both outcomes are logged for weekly review analysis.

### 6. Weekly Review

Generated every Sunday (cron trigger) or on-demand. Claude receives the week's aggregated data and returns a structured coaching report:

- **Wins** — tasks completed, revenue milestones, streaks
- **Failures** — skipped check-ins, low-score task completions, mission switch attempts
- **Revenue generated** (actual logged + task-score estimate)
- **Hours spent** (sum of available_hours from daily check-ins)
- **Focus score** (0–100, weighted by task score completion rate)
- **Completion score** (0–100, primary task completion rate)
- **AI coaching** — direct, specific recommendations for next week

Weekly review is saved to the `weekly_reviews` table and emailed via Resend.

### 7. Momentum Dashboard

The home screen. Displays:
- Active mission title + days on mission
- Mission progress bar (% complete, user-estimated or AI-inferred)
- Revenue this month (actual logged)
- Tasks completed (this mission)
- Focus score + streak
- Today's 3 tasks (primary / secondary / optional) with revenue score badges
- AI Chief of Staff nudge (a direct 1–2 sentence coaching message)
- Quick actions: Jail Idea / Log Revenue / AI Chat

### 8. AI Chat (Chief of Staff Interface)

A persistent chat tab with Claude. Every message includes a system prompt containing full mission context:

```
Current mission: {title} — Day {n}
Today's tasks: {tasks} | Completed: {count}
Revenue this month: ${total}
Focus score: {score} | Streak: {n} days
SAHM mode: {on/off} | Available today: {minutes} min
Recent jailed ideas: {list}
Behavior: Challenge excuses. Celebrate execution. Never suggest new directions unprompted.
```

Chat messages are persisted to the `chat_messages` table scoped to the active mission. The anti-shiny object challenge is delivered through this interface.

### 9. SAHM Mode

A toggle in Settings. When enabled:

- Check-in captures available time in **minutes** (not hours)
- Time selector shows: 15 min / 30 min / 45 min / 1 hr / More
- AI generates **exactly 1 task** — the highest revenue-score action completable in the window
- System prompt injects: `"User has {n} minutes. Give one task only. No list. No options."`
- All coaching language adjusts to respect time scarcity without guilt
- SAHM badge shown in top nav when active

```sql
updated_at       timestamptz DEFAULT now()
```

```sql
id               uuid PRIMARY KEY DEFAULT gen_random_uuid()

title            text NOT NULL
```

```sql
user_id                uuid REFERENCES profiles NOT NULL

description            text
status                 text CHECK (status IN ('jailed','flagged','approved','promoted','dismissed'))

ai_notes               text

created_at             timestamptz DEFAULT now()
```



## AI Agent Workflows

### Daily Check-in Flow
1. User opens app (route guard: redirects to `/app/checkin` if no check-in today)
2. User fills form: available time, obstacles, yesterday's progress
3. `POST /api/checkin` builds prompt with mission context + check-in data
4. Claude returns zod-validated JSON: `{ tasks: Task[], coaching_note: string }`
5. Tasks saved to `tasks` table, check-in saved to `daily_checkins`
6. Dashboard renders tasks

**SAHM override:** If `available_hours < 1.0`, system prompt instructs Claude to return exactly 1 task.

### Idea Jail Flow
1. User submits idea via form or quick action
2. `POST /api/ideas` sends idea + current mission to Claude
3. Claude returns: `{ alignment_score: number, status: string, ai_notes: string }`
4. Score 0–4 → `jailed`, 5–7 → `flagged`, 8–10 → `approved`
5. Idea saved with status and AI notes
6. If `approved`, user is prompted: "This supports your mission. Add it as a task?"

### Anti-Shiny Object Protocol
**Triggers:**
- User navigates to `/app/mission/new` (mission age < 14 days)
- User clicks "Abandon Mission"
- User submits idea with alignment score ≥ 8 AND clicks "Make this my new mission" (explicit pivot attempt)

**Flow:**
1. API route intercepts action before DB write
2. Claude generates challenge message (streamed to chat)
3. User chooses: "Switch anyway" or "Continue mission"
4. Both outcomes logged to `missions.abandoned_reason` or `ideas` status

### Weekly Review Flow
1. Sunday cron job (or user triggers manually)
2. DB aggregates: tasks completed, revenue entries, hours from check-ins, streak data
3. `POST /api/review/weekly/generate` sends stats + mission context to Claude
4. Claude returns: `{ wins: string[], failures: string[], coaching: string, focus_score: number, completion_score: number }`
5. Saved to `weekly_reviews`, emailed via Resend

### Chat System Prompt (injected on every request)
```
You are the user's AI Chief of Staff. You are direct, honest, and relentlessly focused on execution over planning.

Current mission: {mission.title} — Day {days_on_mission}
Today's tasks: {task_titles} | Completed today: {completed_count}/{total_count}
Revenue this month: ${revenue_total} | Focus score: {focus_score}
Current streak: {streak} days
SAHM mode: {enabled/disabled} | Available time today: {available_minutes} min
Recently jailed ideas: {recent_idea_titles}

Behavior rules:
- Challenge excuses directly and specifically
- Celebrate execution, not planning
- Never suggest switching missions or starting new projects
- When user asks about a new idea, tell them it is going to jail
- Be supportive of constraints (time, energy, circumstances) but not of avoidance
- Responses should be short, direct, and actionable
```

## API Structure

| Method | Route | Auth | Purpose |
|--------|-------|------|---------|
| `POST` | `/api/checkin` | required | Submit check-in → Claude generates tasks (zod-typed) |
| `POST` | `/api/ideas` | required | Submit idea → Claude scores alignment (0–10) |
| `PATCH` | `/api/ideas/[id]` | required | Update idea status (promote, dismiss) |
| `POST` | `/api/tasks/[id]/complete` | required | Mark task done, update streak |
| `POST` | `/api/tasks/[id]/skip` | required | Skip task with optional reason |
| `POST` | `/api/revenue` | required | Log a revenue entry |
| `GET` | `/api/review/weekly` | required | Fetch weekly review (query: `?week=YYYY-MM-DD`) |
| `POST` | `/api/review/weekly/generate` | required | Trigger Claude to generate weekly review |
| `POST` | `/api/mission` | required | Create new mission (enforces one-active constraint) |
| `PATCH` | `/api/mission/[id]` | required | Update mission status (complete, abandon + reason) |
| `POST` | `/api/chat` | required | Streaming chat with full mission context |
| `GET` | `/api/profile` | required | Fetch profile + SAHM settings |
| `PATCH` | `/api/profile` | required | Update profile, SAHM mode, time budget |
| `POST` | `/api/cron/weekly-review` | cron secret | Vercel cron job: aggregate week + generate review (Sunday 8am) |

All structured endpoints return zod-validated responses. `/api/chat` returns a `ReadableStream` for streaming.

## User Stories

### Onboarding
- As a new user, I can sign up and immediately set my first mission so I start focused from day one
- As a user, I can enable SAHM mode during onboarding so the app respects my time constraints from the start

### Daily Check-in
- As a user, I can complete a daily check-in in under 2 minutes so I spend time executing, not planning
- As a SAHM user, I can enter "30 minutes" and receive exactly 1 high-leverage task instead of an unrealistic list
- As a user, I can see my tasks for the day immediately after check-in without navigating elsewhere

### Idea Jail
- As a user, I can submit a new idea at any time so it doesn't disappear, but it doesn't derail my focus
- As a user, I can see the AI's honest assessment of whether my idea supports my current mission
- As a user, I can review jailed ideas from past missions without them cluttering my active jail

### Anti-Shiny Object Protocol
- As a user who tries to abandon a mission under 14 days old, I receive a direct challenge before the switch happens
- As a user, I can still choose to switch after being challenged, but the reason is logged for my weekly review

### Revenue Tracking
- As a user, I can log actual revenue earned so my weekly review reflects real progress
- As a user, I can see which task types historically generate the most revenue so I know where to focus

### AI Chat
- As a user, I can ask my AI Chief of Staff anything and receive a response that knows my exact mission context
- As a user, when I mention a new idea in chat, the AI tells me it's going to jail instead of entertaining it

### Weekly Review
- As a user, I receive a weekly coaching report that tells me the truth about my performance, not just praise
- As a user, I can view past weekly reviews to track my progress over time

### SAHM Mode
- As a SAHM, when I have 30 minutes before my baby wakes up, I see one task I can actually complete
- As a SAHM, the AI never makes me feel guilty about limited time — it just maximizes the time I have
